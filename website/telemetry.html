Telemetry

Observable by default. OpenTelemetry native. Every transaction generates logs, metrics, and traces automatically.

The Three Pillars

Ergon provides a production-grade observability stack built entirely on OpenTelemetry standards:

üìù Logging

Structured logs with automatic context propagation.

  • JSON output for log management platforms
  • Auto-injection of trace_id and span_id
  • Console, File, RotatingFile, OTLP handlers

üìà Metrics

Application-level metrics via the OTel MeterProvider.

  • Push-based via PeriodicExportingMetricReader
  • Counters, Histograms, Gauges
  • Auto resource attributes (task, host, PID)

üîç Tracing

Distributed tracing via the OTel TracerProvider.

  • Automatic spans for task, connector, service
  • Hierarchical: Task ‚Üí Batch ‚Üí Transaction ‚Üí Attempt
  • Export to Jaeger, Tempo, any OTLP backend

Configure Once, Instrument Everywhere

The framework handles telemetry initialization automatically:

  • Runners initialize telemetry once per worker at startup
  • Tasks, Connectors, Services request loggers/tracers via standard accessors
  • Configuration is declarative‚Äîswitch backends without code changes

Configuration Example

from ergon_framework.telemetry import logging, tracing, metrics

config = TaskConfig(
    name="order-processor",
    task=OrderProcessorTask,
    connectors={...},
    
    # Logging
    logging=logging.LoggingConfig(
        level="INFO",
        handlers=[
            logging.ConsoleLogHandler(),
            logging.OTLPLogHandler(endpoint="http://collector:4317"),
        ]
    ),
    
    # Tracing
    tracing=tracing.TracingConfig(
        processors=[
            tracing.SpanProcessor(
                processor=tracing.BatchSpanProcessor,
                exporters=[tracing.OTLPSpanExporter(endpoint="http://collector:4317")]
            )
        ]
    ),
    
    # Metrics
    metrics=metrics.MetricsConfig(
        readers=[
            metrics.MetricReader(
                reader=metrics.PeriodicExportingMetricReader,
                exporters=[metrics.OTLPMetricExporter(endpoint="http://collector:4317")]
            )
        ]
    ),
)

Automatic Resource Injection

Every LogRecord, Metric, and Span is automatically tagged with:

Service Name The task name (e.g., OrderProcessor)
Instance ID Unique execution ID of the runner
Process ID PID of the worker
Host Name Machine or pod hostname
Host IP IP address of the host
Transaction ID Unique ID of the current transaction
