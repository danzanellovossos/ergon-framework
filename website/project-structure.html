Shared Modules

Files at the tasks/ root are shared across all tasks:

File Purpose
settings.py Connector configs, service configs, telemetry configs. Single source of truth.
constants.py Enums, thresholds, batch sizes. No magic values scattered in code.
schemas.py Data models used by multiple tasks. DRY principle.
exceptions.py Exception classes for consistent error categorization.
helpers.py Pure utility functions. Testable in isolation.

Per-Task Modules

Each task gets its own directory with the same file pattern:

File Purpose
task.py The task class. Pick ConsumerTask, ProducerTask, or HybridTask.
config.py TaskConfig with connectors, services, policies. Register with task_manager.
schemas.py Data models specific to this task. Import shared from parent.
exceptions.py Task-specific exceptions. Extend shared when needed.
helpers.py Task-specific utilities. Compose with shared helpers.

Quick Reference

Adding a New Task

  1. Create folder: tasks/my_task/
  2. Create files: task.py, config.py, schemas.py, exceptions.py, helpers.py
  3. Pick your task class (AsyncConsumerTask, etc.)
  4. Wire up in config.py
  5. Register with task_manager.register()

Adding a Custom Connector

  1. Create folder: connectors/my_connector/
  2. Create service.py with protocol logic
  3. Create connector.py wrapping the service
  4. Add ConnectorConfig in settings.py
  5. Reference in task config

Using a Plugin

  1. Install: pip install ergon-kafka
  2. Import in settings.py
  3. Configure with ConnectorConfig
  4. Reference in task config

Why This Structure Works

  • Separation of Concerns: Tasks, connectors, and shared modules in separate directories
  • DRY: Global modules prevent copy-paste across tasks
  • Single Responsibility: Each file does one thing
  • Testability: Pure functions in helpers, isolated business logic
  • Discoverability: Consistent naming‚Äîyou always know where to look
  • Scalability: Adding a new task is just copying the pattern
