Connectors

The transaction gateway. Connectors bridge external systems to the framework's transaction interface.

What is a Connector?

A Connector is a structured wrapper around a Service that adapts it to the framework's transaction flow interface. It has exactly two responsibilities:

  1. Fetch Transactions: Pull data from the service, wrap each unit in a Transaction object.
  2. Dispatch Transactions: Accept Transaction objects, extract payloads, send via the service.

Connector Interface

Every connector implements this interface:

from abc import ABC, abstractmethod
from typing import List

class Connector(ABC):
    
    @abstractmethod
    def fetch_transactions(self, batch_size: int, **kwargs) -> List[Transaction]:
        """Fetch and wrap raw data into Transactions."""
    
    @abstractmethod
    def dispatch_transactions(self, transactions: List[Transaction], **kwargs):
        """Unwrap Transactions and send via the service."""
    
    def get_transactions_count(self, **kwargs) -> int:
        """Optional: Get pending transaction count."""

AsyncConnector provides the same interface with async methods.

Composition Pattern

A central design principle:

  • Every Connector holds exactly one Service instance
  • The Connector delegates low-level operations to that Service
  • The Connector adapts the Service's capabilities into fetch_transactions and dispatch_transactions

Service

"I know how to talk to RabbitMQ." ‚Äî Has methods like consume(), ack(), publish().

Connector

"I know how to fetch transactions from RabbitMQ." ‚Äî Calls service.consume(), wraps result in Transaction.

Atomicity Preservation

The connector must never modify atomicity. If the external system defines a batch of 10,000 records as a single indivisible unit, the connector must surface that batch intact as a single Transaction object.

Configuration

Connectors are configured declaratively:

from ergon_framework.connector import ConnectorConfig

INPUT_CONNECTOR = ConnectorConfig(
    connector=RabbitMQConnector,
    kwargs={
        "host": os.getenv("RABBITMQ_HOST"),
        "queue": "orders",
    }
)

Built-in Connectors

Ergon ships with two connectors out of the box:

  • Excel Connector: Read and write Excel files as transactions
  • RabbitMQ Connector: Consume and produce messages from RabbitMQ queues

Additional Connectors

For other transports, you have two options:

  • Install packages: Community or official connector packages available through your SDK's package manager
  • Build custom: Create a connector directly in your project's connectors/ folder without publishing

Custom connectors are perfect for internal APIs or proprietary systems where publishing a package doesn't make sense.
