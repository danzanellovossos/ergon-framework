Services

The protocol experts. Services handle networking, authentication, retries, pagination‚Äîall the messy protocol stuff.

What is a Service?

A Service encapsulates all direct interaction with external systems, APIs, databases, or protocols. They are the "low-level intelligence" of the architecture.

üåê Networking

HTTP, TCP, gRPC, WebSockets‚Äîservices handle the protocol mechanics.

üîê Authentication

API keys, OAuth tokens, session management‚Äîservices own credentials.

üîÑ Resilience

Retries, backoff, circuit breakers, error translation.

üìÑ Pagination

Cursors, offsets, continuation tokens‚Äîservices handle it all.

Standalone and Injected

Unlike Connectors, Services are standalone components. They don't depend on the task framework's transaction loop.

Standalone Usage

Use a service directly in a script or notebook:

service = OpenAIService(api_key="...")
result = service.complete("Hello")

Injected Usage

Inject into tasks for enrichment:

enriched = self.deepl_service.translate(text)

Services vs. Connectors

When a Service is used as the source or sink of a pipeline, it gets wrapped by a Connector. But many services are never wrapped because they're used ad-hoc by business logic:

  • LLM clients (OpenAI, Anthropic) ‚Äî used for enrichment
  • Geocoding APIs ‚Äî used for address lookup
  • Database clients ‚Äî used for queries within tasks

Key Principle

Services do NOT know about Transactions. They work with raw data (dictionaries, strings, bytes). The translation between raw data and Transactions happens in the Connector layer.

Configuration

from ergon_framework.connector import ServiceConfig

OPENAI_SERVICE = ServiceConfig(
    service=OpenAIService,
    kwargs={
        "api_key": os.getenv("OPENAI_API_KEY"),
        "model": "gpt-4",
    }
)

What Services Own

  • Persistent state: Open sessions, sockets, HTTP clients, stream cursors
  • Retry logic: All backoff strategies, throttling, circuit-breaking
  • Error normalization: Translating protocol errors into domain errors
  • Pagination: Tokens, cursors, continuation‚Äîno pagination logic leaks into tasks

Observability

Services receive tracer and logger instances automatically. They annotate tracing spans with protocol details: latency, payload sizes, retry counts, batch sizes, or error categories.
